### 반복자 패턴
컬렉션의 내부구조를 노출하지 않고도 요소를 순차적으로 탐색할 수 있게 해주는 디자인 패턴을 의미한다. 
- Iterator 객체
    - 실제로 순회하는 객체
    - hasNext() : 다음요소 있는지 확인
    - next() : 다음요소 반환

예시
```java
List<String> list = List.of("A", "B", "C");

Iterator<String> it = list.iterator();

while (it.hasNext()) {
    System.out.println(it.next());
}
```

### ::연산자
```java
String[] favorites = req.getParameterValues("favorite");
        System.out.println("favorites...");

//                .forEach(thing -> System.out.println(thing));
```

:: 연산자는 **이미 존재하는 메서드를 람다식 대신 참조해서 사용하는 문법을 의미한다.**
매개변수를 적지 않아도 되어서 가독성이 높아지는 장점이 있다. 이는 그래서 기존 메서드의 호출이나 추가 로직이 없는 경우에 사용하는 것을 추천한다.

### 표준입출력
Stream은 데이터 입출력시 모든 데이터를 형태와 관계없이 일관된 흐름으로 전송하는 것이다.
- 프로그래밍에서의 스트림 종류
    - 파일데이터 : 파일의 시작과 끝의 데이터 스트림
    - http 송수신 데이터: http 헤더, 바디의 데이터들
    - 키보드 입력 : 사용자가 키보드로 입력하는 숫자 및 문자들
그 중 표준 입출력은 java에서는 흔히 System.in, System.out이라고 표시하는 것으로 
사용하고 System.in은 InputStream, System.out은 PrintStream에서 담당한다.
OS가 기본으로 제공한다.

### 출력 연산자, 버퍼
표준입출력에서 사용한 System.out.println(), System.out.print()가 java에서 흔히 사용하는 
출력연산자이다. 이러한 출력을 통해 데이터를 콘솔, 파일, 등으로 보내게 된다. 
하지만 바로 출력을 하게되면 Interrupt에 의한 context switch가 자주 발생하게 되고 이러한 이유로 속도가 느려지게 된다.

이를 위해 데이터를 임시공간에 넣고 한번에 출력하는 쪽으로 방향이 바뀌게 되었다. 이때의 임시공간을 버퍼라고 하고 조건이 되면 한번에 출력하게 된다.
- 버퍼가 출력하는 경우
    - 줄바꿈 발생
    - 버퍼가 꽉찼을 때
    - 명시적으로 flush()함수를 호춣한 경우
    - 프로그램을 종료하는 경우
### EL(Expression Language)
JSP 서버사이드에서 자바 코드를 직접 쓰지 않아도 데이터를 꺼내서 간단한 출력이나 연산을 할 수 있게 해주는 연산식
아래는 Jstl.jsp파일에서 일부 가져온 것이다.
```jsp
<p>
    보내주신 값은?

    ${result}
</p>
```
jsp파일에 값을 넣기 위해 사용한 java코드이다.
```java
req.setAttribute("result", result);
```

EL이 데이터를 찾는 순서
1. pageScope
    현 JSP 페이지 안에서만 사용할 수 있는 영역
2. requestScope
    하나의 http request에서 유지되는 영역, 컨트롤러에서 jsp로 데이터를 넘길때 사용
3. sessionScope
    로그인 정보같은 데이터를 저장할 때 사용
4. applicationScope
    웹 애플리케이션 전체에서 공유되는 영역, 모든 사용자가 같이 접근 가능하다.

    ```jsp
    <%
    pageContext.setAttribute("name", "page");
    request.setAttribute("name", "request");
    session.setAttribute("name", "session");
    application.setAttribute("name", "application");
    %>

    ${name}
    ```
    위 코드에서 같은 변수를 여러 스코프를 주고 찾으려고 하면 가장 먼저 발견한 값으로 return을 한다.

### Jstl
JavaServer Pages Tag Library의 약자 JSP에서 자바 코드를 직접 쓰지 않고도 반복, 조건문, 출력, 포맷들을 태그로 처리할 수 있게 하는 라이브러리
```jsp
<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>
```
위 코드를 위에 선언해야지 사용이 가능하다

반복문 사용
```jsp
<c:forEach var="item" items="${list}">
    ${item}
</c:forEach>
```

if-else문 사용
```jsp
<c:choose>
    <c:when test="${score > 60}">합격</c:when>
    <c:otherwise>불합격</c:otherwise>
</c:choose>
```
EL과 함께 사용되고 java코드를 줄이고 가독성을 향상시키기 위해 사용했다고 볼 수가 있다.

### DSL
Domain Specific Language의 약자로, 특정 분야를 쉽게 표현하기 위해 만든 전용언어를 의미한다.
c, java와 같은 프로그래밍 언어는 범용적이라서 비개발자가 이해하기 어렵다. 그래서 특정분야만 쉽게 표현하기 위해 사용한다.
- 예시
    - sql : 데이터베이스 특화
    - html : 웹페이지 구조 특화
    - css : 스타일 표현 특화

이와 같은 DSL은 **"도메인을 얼마나 잘 표현하는가"**가 핵심이라고 볼 수가 있다.
### 랜더링
데이터나 코드로 작성된 내용을 실제 화면에 보이도록 그리는 과정을 의미한다.
대표적으로 서버 사이드 랜더링(SSR), 클라이언트 사이드 랜더링(CSR)이 존재한다.
- SSR
    서버에서 html을 완성해서 보내준다
    대표적으로 jsp, thymeleaf가 존재한다
- CSR
    브라우저에서 JS로 화면을 생성한다.
    대표적으로 React가 있다.
    초기 로딩은 SSR에 비해 느리지만 이후는 빠르다

## Postman으로 패킷받기 실습
postman은 API를 테스트하고 개발할 때 사용하는 대표적인 도구이다.
이것을 통해 아직 배포하지 않고 개발중인 서버에 httpRequest를 보내고 이에 대한 httpResponse를 받아와서 테스트를 진행할 수가 있다.
브라우저에 테스트하는 것은 GET은 쉽지만 POST,PUT, DELETE와 같은 테스트가 어려워서 이 도구를 통해 테스트를 한다고 한다.

```java
method = GET
requestURI = /c01
authType = null
Authorization = Bearer SUPER_SECRET_KEY
User-Agent = PostmanRuntime/7.53.0
Accept = */*
Postman-Token = 22b1ba7b-5cbe-49bc-84d8-b08caa80b47b
Host = localhost:8080
Accept-Encoding = gzip, deflate, br
Connection = keep-alive
Cookie = Cookie_1=value
key= key1value = [val1]
key= key2value = [val2]
key= key3value = [val3]
```
쿠키까지 만들어서 테스트를 해본 결과이다. Authorization 관련은 임의의 키를 만들었지만 이후 JWT를 다루면서 생성해볼 것이다.

# Emmet 활용
---
html을 작성하는데 도움을 주고 특정 키워드만 있으면 긴 html문을 자동 완성시켜준다.


리스트안의 리스트 구현
```html
<!-- div>ul>li^h2+span -->
    <div>
        <ul>
            <li></li>
        </ul>
        <h2></h2>
        <span></span>
    </div>
```

반복자를 사용하고 싶을 때 활용하면 좋을 것 같다.
```html
<!-- ul>li*5 -->
    <ul>
        <li></li>
        <li></li>
        <li></li>
        <li></li>
        <li></li>
    </ul>
```

리스트안의 리스트와 반복문 혼합 사용
```html
<!-- div>(nav>ol>li*3)+div -->    
    <div>
        <nav>
            <ol>
                <li></li>
                <li></li>
                <li></li>
            </ol>
        </nav>
        <div></div>
    </div> 
```
