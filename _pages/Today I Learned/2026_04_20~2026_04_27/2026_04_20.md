# MyBatis configuration 구성
---

### MyBatis-configration 문서의 예시
---
```XML
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration
        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-config.dtd">

<configuration>

    <environments default="exp1">

        <environment id = "exp1">

            <transactionManager type = "JDBC" />

            <dataSource type = "POOLED">

                <property name="driver" value="com.mysql.cj.jdbc.Driver" />
                <property name="url" value="jdbc:mysql://localhost:3306/mybatis_shop" />
                <property name="username" value="rapa_mybatis" />
                <property name="password" value="pwd1" />

            </dataSource>

        </environment>
    </environments>

    <mappers>
        <mapper resource="mapper/TestMapper.xml" />
    </mappers>

</configuration>

```

dataSource에 커넥션을 어떻게 설정할 지에 대해 적는다. 여기서 제공하는 POOLED는 HikariCP가 아닌 mybatis에서 제공하는 자체 커넥션 풀이다. 

# MyBatis란?
---
MyBatis는 SQL 실행 결과를 객체로 자동 변환해주는 대표적인 SQL Mapper 프레임워크이다.
jdbc의 ResultSet 객체로부터 받아오는 값을 객체로 매핑하는 역할을 수행한다. XML에 SQL 쿼리를 직접 작성함에 따라 튜닝할 수 있고 JDBC 코드의 불필요한 반복을 제거할 수 있는 장점이 있다. 

### factory를 활용하여 session 구현
```java
public class SessionFactory1 {
    static void main(String[] args) throws Exception{

        //sqlSessionFactory
        //DB connection Settings
        //실제 쿼리를 실행한다.

        String xmlFileDir = "mybatis-config1.xml";

        InputStream xmlInputStream = Resources.getResourceAsStream(xmlFileDir);

        //MyBatis 전체 설정 정보와 실행환경을 내부에 포함하고 있는 설정 컨테이너 객체
        //SqlSession 객체를 생성하는 객체
        SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(xmlInputStream);

        //실제 작업자
        //SqlSession을 하면서 preparedStatement까지 포함한다
        //SqlSession은 쓰레드세이프를 지원하지 않는다. 이 경우 병렬작업을 시도하는 경우 오류 발생
        //한 쓰레드에 하나의 SqlSession을 사용하도록 할 것 -> tomcat은 요청 하나당 하나의 쓰레드를 생성해서 무관
        //만약 쓰레드를 이벤트마다 생성해주는 경우 예외처리를 잘 작성해야 할 것
        try(SqlSession sqlSession = sqlSessionFactory.openSession()){

            DataSource dataSource = sqlSessionFactory.getConfiguration()
                    .getEnvironment()
                    .getDataSource();

            System.out.println("dataSource" + dataSource.getClass().getName());
            System.out.println("Sql 세션 획득 성공!");

            //namespace.query_id
            Integer result = sqlSession.selectOne("test.test");
            System.out.println("result : "+ result);


        }catch (Exception e){
            throw new RuntimeException(e);
        }

    }
```

### hikariCP로 설정을 변경하고 싶은 경우
---
```xml
<dataSource type = "config.HikariCpDataSourceFactory">

    <property name="driver" value="com.mysql.cj.jdbc.Driver" />
    <property name="url" value="jdbc:mysql://localhost:3306/mybatis_shop" />
    <property name="username" value="rapa_mybatis" />
    <property name="password" value="pwd1" />

</dataSource>
```  
dataSource부분을 설정한 임의의 config파일에 설정한 hikariCP에 저장한 값을 가져올 수 있도록 설정

```java
public class HikariCpDataSourceFactory implements DataSourceFactory {

    private Properties properties;

    @Override
    public void setProperties(Properties props) {
        this.properties = props;
    }

    @Override
    public DataSource getDataSource() {
        HikariConfig config = new HikariConfig();
        config.setDriverClassName(properties.getProperty("driver"));
        config.setJdbcUrl(properties.getProperty("url"));
        config.setUsername(properties.getProperty("username"));
        config.setPassword(properties.getProperty("password"));


        return new HikariDataSource(config);
    }
}
```

## mapper을 활용한 SQL문 등록하기
---
mapper 파일에 원하는 sql문을 작성하여 이것을 가져와 사용하는 것이 가능하다.


resource의 mapper폴더 안의 s2폴더 안에 저장한 MemberMapper.xml파일에 튜닝한 sql문을 작성해놓는다. 단건 조회, 다건 조회와 같이 목적에 맞는 CRUD 작성
```XML
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="mapper.s2.MemberMapper">
    <select id = "findAll" resultType="domain.Member">
        SELECT
            id,
            name,
            password,
            email,
            balance
        FROM
            member
    </select>

    <select id = "findById" resultType="domain.Member" parameterType="long">
        SELECT
            id,
            name,
            password,
            email,
            balance
        FROM
            member
        WHERE
            id = #{id}
    </select>

    <select id = "findByEmail" resultType="domain.Member" parameterType="String">
        SELECT
            id,
            name,
            password,
            email,
            balance
        FROM
            member
        WHERE
            email = #{email}
    </select>

    <insert id = "save" useGeneratedKeys="true" keyProperty="id">
        INSERT INTO member
            (name, password, email, balance)
        VALUES
            (#{name},#{password},#{email},#{balance})
    </insert>

    <update id = "updateBalance">
        UPDATE member
        SET
            balance = #{amount}
        WHERE
            id = #{id}
    </update>

    <delete id = "deleteById" parameterType="long">
        DELETE FROM member
        WHERE
            id = #{id}
    </delete>

<!--    resultType 체크-->
    <select id = "count" resultType="Integer">
        SELECT
            COUNT(*)
        FROM
            MEMBER
    </select>

</mapper>
```

#### mapper 패키지에 저장한 interface MemberMapper interface를 활용
mapper 패키지에 Member 테이블에 관해 MemberMapper을 만들어 interface를 활용한 프록시 객체를 만들 수가 있다.
```java
public interface MemberMapper {
    //CRUD
    //다건조회
    List<Member> findAll();

    //단건조회
    Optional<Member> findById(Long id);
    Optional<Member> findByEmail(Long Email);

    int save(Member member);
    int updateBalance(@Param("id")Long id,@Param("amount") int amount);

    int deleteById(Long id);

    // 총 회원수
    int count();

}
```

#### mybatis를 적용한 member 구현
앞의 설정한 mapper와 domain에 입력한 테이블의 정보를 가져와 구현
```java
public class MemberCrud {
    static void main() throws Exception {
        MyBatisSessionConnector connector = new HikariMyBatisConnector(
                "mapper.s2",
                "domain"
        );


        try(SqlSession connection = connector.openSession(true)){
            MemberMapper memberMapper = connection.getMapper(MemberMapper.class);

            Member member1 = new Member(
                    null, "user_" + genRandomNumber(), "1234", "user%d@email.com".formatted(genRandomNumber()), genRandomNumber()
            );

            Member member2 = new Member(
                    null, "user_" + genRandomNumber(), "1234", "user%d@email.com".formatted(genRandomNumber()), genRandomNumber()
            );

            memberMapper.save(member1);
            memberMapper.save(member2);

            System.out.println("member1 id = "+ member1.getId());
            System.out.println("member1 id = "+ member2.getId());
        }
    }
}
```
앞서 작성한 interface를 통해 작성한 프록시 객체를 활용함과 Dependency injection을 통해 유지보수가 쉽도록 구현했다. 

try with resources문을 만들어 close문을 작성하지 않아도 sql 연결에 소모되는 자원을 안전하게 반납하도록 구현했다.

여기서 connector객체는 jdbc의 트랜잭션의 기능을 구현한다. 이때 만약 connector의 openSession(true)를 하지 않고 openSession()을 하면 기본값이 false로 되어있어 데이터베이스에 commit이 적용이 되지않아 insert, update, delete 작업은 반영이 되지 않는다.



# 예제에서 사용한 몇가지 디자인 패턴정리
---
디자인 패턴은 수많이 반복되는 설계 문제에 대해 검증된 해결방법으로 코드를 구조적으로 분리하고 확장성을 증가시켜 구현할 수 있고 협업 시 소통을 원활하게 할 수 있으므로 정리가 필요하다.
## Builder
---
객체 생성 과정이 복잡할 때, 가독성과 안정성을 높이기 위해 단계적으로 객체를 생성하는 패턴
이때 파리미터의 순서는 신경쓰지 않아도 되는 경우에서 사용하면 좋다. 



## Proxy
---
실제 객체 대신 대리 객체를 사용하여, 접근을 제어하거나 공통 기능을 추가한 패턴이다. Spring의 AOP(관점지향 프로그래밍)이나 트랜잭션 처리에 핵심적으로 사용이 된다.

```java
public interface MemberMapper {
    //CRUD
    //다건조회
    List<Member> findAll();

    //단건조회
    Optional<Member> findById(Long id);
    Optional<Member> findByEmail(Long Email);

    int save(Member member);
    int updateBalance(@Param("id")Long id,@Param("amount") int amount);

    int deleteById(Long id);

    // 총 회원수
    int count();

}
```
위 코드처럼 인터페이스를 활용하여 객체에 부가기능을 더하여 구현
