# 네이버 토큰을 가져오는 OAuth2.0 방식 실습
---
![naver_login_resource](/assets/img/naver_login_resource.png)
네이버도 카카오처럼 Spring에서 지원해주지는 않으므로 등록시 authorization-grant-type, client-authentication-method, redirect-uri등을 추가로 설정해줘야 한다.
그리고 별도로 **provider**를 만들고  **authorization-uri,  user-name-attribute, token-uri,  user-info-uri**을 넣어 주어야 한다.

카카오와 다소 다른점은 위 사진처럼 response 객체로 받고 response 객체 하위에 접근하여 유저정보를 받아보는 식으로 진행된다. 그래서 OAuth2.0 방식으로 유저에 관한 데이터를 받아오는 경우 공식 API문서를 참조를 하는 편이 좋을 것 같다.

# 스웨거 Swagger
---
[스웨거 에디터 링크](https://editor.swagger.io/)
이렇게 외부 프로그램과 연결하거나 프로그램에 관해 작성할 때 API문서 형식으로 작성을 하게 된다. 이럴 때 자주사용하는 것으로는 swagger API가 있는데 REST API를 문서화하고 테스트할 수 있게 도와주는 도구라고 볼 수가 있다. 

- 사용 이유
1. 코드를 기반으로 API 문서 생성 가능
2. 브라우저에 직접 요청하여 API 테스트 가능
3. Request/Response 확인
4. 프론트엔드와 백엔드 간의 협업에서 효율이 증가

![swagger_edit](/assets/img/swagger_edit.png)


```java
implementation 'org.springdoc:springdoc-openapi-starter-webmvc-ui:2.8.6'
```
위 의존성을 추가하면 spring이 Controller을 분석하여 API를 읽어드려 자동으로 문서를 생성한다.
그리고 Swagger Annotation을 붙여 자세한 문서를 만드는 것이 가능하다.

- @Operation : API 설명
- @Parameter : 파라미터 설명
- @Schema : DTO 설명
