### Result 출력 

```java
private static int getStock(Connection conn, Long productId) throws SQLException {
    String sql = "select stock from product where id = ?";
    try(PreparedStatement stmt = conn.prepareStatement(sql)){
        stmt.setLong(1, productId);
        try(ResultSet rs = stmt.executeQuery()){
            return rs.next() ? rs.getInt("stock") : 0;
        }
    }
}
```

### 주의 SQL injection

임의의 SQL문을 주입하고 실행되게 하여 취약점 침입을 하는 것
- SQL injection 예시
```SQL
select
    *
from
    member
where
    email = 'user@email'and password = '' OR '' = '';

```
위와 같이 비밀번호를 TRUE로 만들어주는 값을 대입하여 사용자 정보를 탈취하는 것이 가능하다.


### 커넥션 풀
**배경** : JBDC 연결은 드라이버를 로드하고 연결하여 객체를 받아와야 하는 과정을 거치게 되는데 이때 되게 많은 시간과 자원소모에 대한 단점을 가지게 된다.


**개념** : 최초 Pool 내에서 Connection들을 하여 HTTP 요청에 따라 응답을 제공하고 반환받아서 이를 재사용하는 것

**과정 **
1. 애플리케이션이 실행되면서 POOL 내에 connection들을 생성해둔다.
2. HTTP의 요청이 올 때 Pool 내에서 Connection 객체를 가져다 사용
3. 사용이 완료된 Connection 객체는 Pool 내에 반환된다.

## scope 공부
변수나 메서드, 클래스가 어디서부터 어디까지 접근 가능한지 정의하는 범위


클래스 스코프
- 클래스 스코프는 클래스 전체에서 접근할 수 있는 변수나 메소드의 범위를 의미

메서드 스코프
- 메소드 내에서 선언된 변수들이 가지는 범위
- 메소드가 종료될 때 소멸된다.
- 메소드 내부에서 접근 가능하며, 메소드가 호출될 때마다 새롭게 생성

블록 스코프
- {}로 묶인 코드 블록내에서 선언된 변수들이 가지는 범위
- 변수들은 해당 블록 내에서만 유효, 블록을 벗어나면 소멸
